# 其他

之前的几个篇幅介绍了 thexp 中的试验框架，你可以单独使用其中的某项功能，也可以通过 Trainer 或者其他你愿意的方式对其进行组合。 十分推荐结合 Experiment 来记录你试验中的每一个细节，你将能够更清晰的组织你的目录，查看你的试验。

而此外，作为一个工具包，thexp 还提供了许多小的功能，更多的小功能会在接下的很长一段时间内持续更新，这取决于我试验中遇到的拖慢我试验效率的部份和项目issue。由于精力问题，在写完注释后，暂时没有精力对所有功能逐一介绍，这里只列举最好用的N个，有兴趣的同学欢迎阅读源码。

# 自动分配 device

使用GPU训练是常态了，对每一个向量，都使用 .to(device) ，虽然并不十分繁琐，但确实没有必要。最终在阅读了 torch 相关代码后，我发现 Dataloader 接收一个参数 collate_fn，用于将 sample 级别的数据转换为 batch 级别的数据。因此我修改了该参数的默认值，构成了 AutoCollate：





 





from thexp.torch.data.collate import AutoCollate
from torch.utils.data.dataloader import DataLoader
import torch
device = torch.device('cuda:1')
collate_fn = AutoCollate(device)
dataloader = DataLoader(...,collate_fn=collate_fn)
...
model.to(device)

# 目录组织

thexp 的各模块按照如下的方式进行区分：

  • analyse : 用于对基于 Experiment 完成的试验进行分析和可视化
  • base_classes : 提供了很多基类和好用的数据结构
  • calculate : 提供了很多用于向量、显存、数值计算的功能
  • frame : 所有介绍的框架相关功能均位于该模块内
  • future : 以后打算添加但由于各种原因还没有真正添加进去的功能
  • torch : 对pytorch中许多功能的进一步增强
  • util : 一些基本的功能库
