# 试验流程抽象：Trainer

该模块位于 thexp.frame.trainer 是一套整合了之前提到的各模块，添加了回调机制，以及抽象出了完整训练逻辑的一个模块，主要类为 Trainer。

Trainer 类内部维护了之前提到的中的 Saver/RndManager/Logger/Experiment 变量，同时还维护了 Tensorboard中的 SummaryWritter，这些方式均通过 @property 注解实现了懒加载，在使用到时才会创建，并在 Experiment 中进行相关插件的注册。

# 内部维护变量

在下面的示例中可以看到 Trainer 维护的这些变量，其中，saver/rnd/logger/writter 均使用懒加载，仅在使用时会请求 experiment 分配目录和生成文件。

from thexp import Trainer,Params
import random

class myTrainer(Trainer):
    pass

trainer = myTrainer(Params())

for i in range(50):
    trainer.logger.info(i)

for i in range(20):
    trainer.writter.add_scalar("test",random.random(),i)

# 完整训练流程

下面展示了一个简单的完整的训练流程

模型部份

import torch.nn as nn
class MyModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(28 * 28, 10)

    def forward(self, x):
        x = x.view(x.shape[0], -1)
        x = self.fc(x)
        return x

Trainer部份：定义一个类，并重写 callbacks/datasets/models/train_batch 方法

from thexp.frame import Meter, Params, Trainer
class MyTrainer(Trainer):
    def callbacks(self, params: Params):
        from thexp import callbacks
        callbacks.LoggerCallback().hook(self) # 添加日志输出功能

    def datasets(self, params: Params):
        from torchvision import transforms
        from torchvision.datasets import FakeData
        from thexp.torch.data.dataloader import DataLoader

        dataset = FakeData(size=32 * 10, image_size=(28, 28), transform=transforms.ToTensor())
        train_loader = eval_loader = test_loader = DataLoader(dataset, shuffle=True, batch_size=32, drop_last=True)

        self.regist_databundler(
            train=train_loader,
            test=test_loader,
            eval=eval_loader,
        )

    def models(self, params: Params):
        from torch.optim import SGD
        self.model = MyModel()
        self.optim = SGD(self.model.parameters(), lr=params.lr)
        self.cross = nn.CrossEntropyLoss()

    def train_batch(self, eidx, idx, global_step, batch_data, params, device):
        optim, cross = self.optim, self.cross
        meter = Meter()
        xs, ys = batch_data

        # 训练逻辑
        logits = self.model(xs)
        meter.loss = cross(logits, ys)

        # 反向传播
        meter.loss.backward()
        optim.step()
        optim.zero_grad()

        return meter


params = Params()
params.epoch=20
params.lr = 0.01

trainer = MyTrainer(params)
trainer.train()

# 更改试验名

上面的日志中，可以看到实验级别的目录名为 __main__.myExp 。Trainer中维护了一个类变量 __exp_name__，其默认值为小写化的类名，且 trainer 被替换成 Exp。Trainer中实验名的构造方法为 {模块名}.{__exp_name__}。

如果需要主动命名实验，则需要更改该类的类变量 __exp_name__：，此时实验名构造为 {模块名}.{__exp_name__}

class MyTrainer(Trainer):
    __exp_name__ = "DemoExp"

此时构造的实验名就会变成

TIP

添加模块名是为了尽可能的保证实验名唯一。请尽可能的对各实验采用不同的，易于区分的命名方式。

# 回调函数

thexp 为 Trainer 提供了一套回调机制，在之前的示例中，MyTrainer类重写了 callbacks() 方法，并在其中注册了 LoggerCallback，从而在训练过程中能够自动完成日志输出。如果不注册该回调类，则在大多数过程中程序都会保持静默。

TIP

在 Trainer 的一些方法中，也有一些方法会主动的进行日志输出。

所有回调类的注册方法均通过 hook(trainer) 来完成，thexp提供了众多的回调函数，包括自动保存、日志自动输出等，位于 thexp.frame.callbacks，但可以通过下面的语句直接导入：

from thexp import callbacks
